Muhammad Biograohy (Jekyll + Chirpy)
-------------------------------------------------

Core facts
- Static site built with Jekyll 4.3 using the Chirpy theme (7.x).
- Ruby + Bundler are required; GitHub Pages-compatible stack.
- Key folders: _posts/, _tabs/, assets/img/, assets/img/posts/, Projects _config.yml, Gemfile, index.html.

Prerequisites (Windows-friendly)
1) Install Ruby (3.1+ recommended) with DevKit and ensure `ruby` and `bundle` are on PATH.
2) Install Git (for clone/publish).
3) Optional: `ridk install` to add MSYS2 toolchain if Ruby prompts for it.

Recreate the repository structure
1) Create an empty folder ().
2) Add the following files/directories:
   - Gemfile (content below)
   - _config.yml (content template below)
   - index.html (front matter only: `---\nlayout: home\n---`)
   - _posts/ (Markdown posts with `YYYY-MM-DD-title.md`)
   - _tabs/ (about.md, archives.md, categories.md, tags.md)
   - assets/img/ with subfolder assets/img/posts/ for post images.
3) Keep `_site/` out of version control (Jekyll output).

Gemfile content
```
source "https://rubygems.org"

gem "jekyll-theme-chirpy", "~> 7.0", ">= 7.0.1"

gem "jekyll", "~> 4.3.0"
gem "html-proofer", "~> 5.0", group: :test

platforms :mingw, :x64_mingw, :mswin, :jruby do
  gem "tzinfo", ">= 1", "< 3"
  gem "tzinfo-data"
end

gem "wdm", "~> 0.1.1", :platforms => [:mingw, :x64_mingw, :mswin]
gem "http_parser.rb", "~> 0.6.0", :platforms => [:jruby]

group :jekyll_plugins do
  gem "jekyll-paginate"
  gem "jekyll-redirect-from"
  gem "jekyll-seo-tag"
  gem "jekyll-archives"
  gem "jekyll-sitemap"
  gem "jekyll-feed"
end
```

_config.yml essentials (adjust as needed)
```
theme: jekyll-theme-chirpy
lang: en
timezone: Asia/Karachi
paginate: 10


plugins:
  - jekyll-paginate
  - jekyll-redirect-from
  - jekyll-seo-tag
  - jekyll-archives
  - jekyll-sitemap
  - jekyll-feed
sass:
  style: compressed
compress_html:
  clippings: all
  comments: all
  endings: all
  profile: false
  blanklines: false
  ignore:
    envs: [development]
jekyll-archives:
  enabled: [categories, tags]
  layouts:
    category: category
    tag: tag
  permalinks:
    tag: /tags/:name/
    category: /categories/:name/
```

Setup steps (fresh machine)
1) Run `bundle install` in the project root.
2) Start local server: `bundle exec jekyll serve` then visit http://localhost:4000/.
3) Build static output (optional): `bundle exec jekyll build` (outputs to `_site/`).

Writing posts
- Place new Markdown files in _posts/ with `YYYY-MM-DD-title.md` naming.
- Include front matter:
```
---
title: Post Title
date: YYYY-MM-DD HH:MM:SS -0500
categories: [Category1, Category2]
tags: [tag1, tag2]
---
```
- Images: put in assets/img/posts/ and reference with `/assets/img/posts/<file>`.

Tabs pages
- _tabs/about.md, _tabs/archives.md, _tabs/categories.md, _tabs/tags.md use `layout: page` and `permalink: /:title/` (already set in defaults). Update content as needed.

Deploying
- For GitHub Pages: set `url` in _config.yml to the live domain, push main branch, and enable Pages from the repo settings. Chirpy works with GitHub Pages-compatible gems listed above.

Minimal sanity checks
- `bundle exec jekyll doctor` to detect common issues.
- `bundle exec htmlproofer ./_site` after a build if you want broken-link checks.

If everything is lost and you ask AI to rebuild
- Provide this playbook plus the desired site title/tagline, timezone, and social links.
- Ask it to recreate Gemfile, _config.yml (using values above), index.html front matter, folder skeleton, and sample post front matter.
- After files are rebuilt, run `bundle install` then `bundle exec jekyll serve` to verify.
